# Правила работы с проектом

## Структура файлов

Каждая страница — отдельный HTML-файл. Весь код (HTML + CSS + JS) находится внутри одного файла. Никаких внешних библиотек, фреймворков, CDN и npm.

## Дизайн

- Фон: тёмный (близко к `#0f0f1a`)
- Текст: светлый (близко к `#f0e6ff`)
- Акцентный цвет: оранжевый (например `#f97316`)
- Скруглённые углы, карточки, градиенты — в стиле уже существующих страниц
- Никаких внешних шрифтов — только системные (`Segoe UI`, `system-ui`, `sans-serif`)

## Адаптивность

Все страницы должны хорошо выглядеть на мобильных устройствах:
- Mobile-first подход
- Flexbox / CSS Grid для раскладки
- Кнопки и ссылки — достаточно крупные для нажатия пальцем (min 44px)
- Проверять вёрстку при ширине от 360px

## Стиль текста

- Простой и дружеский язык — как будто объясняешь знакомому
- Без канцелярита: не «осуществить», а «сделать»; не «в целях», а «чтобы»
- Короткие предложения, конкретные формулировки
- Обращение на «вы» (с маленькой буквы)

## Перед созданием нового файла

Сначала описать, что планируется сделать: название файла, назначение страницы, из каких блоков будет состоять. Только после подтверждения — приступать к работе.
